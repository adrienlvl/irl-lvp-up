# #286 — Bilan de sommeil de la semaine (1.9.220)

**Rotation 19 · item #3 · liberté totale (domaine : bien-être / récupération)**

## Problème
Le sommeil n'apparaissait dans l'app que de façon conditionnelle, via la note
de dette de sommeil (`sleepDebtHours`) quand un déficit existait. Aucun repère
positif ni synthèse hebdomadaire : impossible de voir d'un coup d'œil « est-ce
que je dors assez cette semaine ? ».

## Amélioration
Nouveau bandeau **bilan de sommeil hebdo** dans le panneau Récupération :
moyenne d'heures/nuit sur 7 jours glissants + nuit la plus courte + statut
coloré (court / bon / excellent).

### Logique pure — `weeklySleepStats(recovery, startKey, endKey)`
- Filtre les entrées `recovery` valides (`YYYY-MM-DD`) dans la fenêtre `[start, end]`.
- Déduplique par date (dernière valeur > 0 conservée par date).
- Renvoie `{ nights, avg (arrondi 0,1), min, status }` ou `null` si aucune nuit.
- Statut : `avg < 7 → court` · `7 ≤ avg ≤ 8,5 → ok` · `> 8,5 → bon`.

### Rendu — `renderWeeklySleep()`
- Fenêtre = 7 derniers jours (`dateKey(now-6j)` → `localDate()`).
- Masqué si aucune donnée. Classe `weekly-sleep wsl-<status>` (bordure gauche
  ambre / bleu / vert). Nombres en virgule décimale FR.
- Câblé dans `render()` après `renderAthlete()`.
