# Boucle #176 (autonome) — Garde-fou CSS automatique · build 1.9.110

**Contexte :** thème E — après avoir corrigé le bug de parenthèse dans la largeur des dialogues (#175), empêcher ce type de régression de revenir.

## Livré

Un **test automatique** `test/css-lint.test.js` (ajouté à `npm run verify`) qui, à chaque vérif :

1. scanne **tous les fichiers `.css`** et **échoue si une déclaration `propriété:valeur` a des parenthèses déséquilibrées** (le cas exact du bug `width:min(440px,calc(100% - 32px)`), en ignorant proprement les `@media(...)` ;
2. vérifie que **chaque fichier a ses accolades équilibrées**.

Un scan préalable a confirmé qu'il ne reste **aucun autre déséquilibre** dans les 17 fichiers CSS. Ce filet évite qu'une future édition CSS ne casse silencieusement une règle (déclaration ignorée par le navigateur).

## Détail technique

- `src/test/css-lint.test.js` : 3 tests node:test (présence de CSS, parenthèses équilibrées, accolades équilibrées), lecture des `../*.css`, commentaires strippés.
- `package.json` : script `test` étend la liste des fichiers node:test avec `test/css-lint.test.js`.

## Vérifs

- `npm run verify` → **216 tests / 216 pass** (+3 : garde-fou CSS). **SMOKE OK**. `node --check app.js` OK.
- `npm run dist` → **Setup 1.9.110.exe** (app d'Adrien jamais fermée).
