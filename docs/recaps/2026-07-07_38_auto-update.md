## Demande d'Adrien
Question : « le .exe se met à jour automatiquement ? » → réponse : non. Choix d'Adrien : **« Mettre en place l'auto-update ».**

## Ce qui a été fait (code branché, prêt à activer)
- **`electron-updater`** ajouté en dépendance runtime + config `build.publish` (GitHub) dans `package.json` (owner/repo en **placeholder** pour l'instant).
- **`electron-main.cjs`** : `initAutoUpdate()` — **seulement en build empaqueté** — vérifie GitHub Releases au démarrage (à +3,5 s), télécharge en fond, envoie l'état au renderer. **Toutes les erreurs avalées** (dépôt non configuré / hors-ligne) → l'app reste 100 % locale et fonctionnelle. IPC `update:install` (quitAndInstall) + `update:check`.
- **`preload.cjs`** : `onUpdateStatus(cb)`, `installUpdate()`, `checkForUpdate()`.
- **Renderer** : bannière discrète `#updateBanner` → « disponible / téléchargement X % / prête », bouton **Redémarrer & installer**, bouton masquer. Rien affiché en cas d'erreur (dépôt non configuré).
- **Sécurité** : seul le process principal contacte GitHub (HTTPS, hôte fixe). Le renderer garde son verrouillage total (navigation/fenêtres bloquées, CSP `script-src 'self'`). Cohérent avec la Vague S.
- **Build** : `latest.yml` (flux) + `app-update.yml` (embarqué) générés. Script **`npm run release`** ajouté (`electron-builder --win -p always`).
- **Doc** : `docs/AUTO-UPDATE.md` — étapes d'activation.

## ⚠️ À faire par Adrien pour l'activer (une seule fois)
1. Créer un **dépôt GitHub** (public = plus simple).
2. Renseigner `owner`/`repo` dans `package.json` → `build.publish`.
3. Créer un **token GitHub** (portée `repo`).
4. `$env:GH_TOKEN="…"; npm run release` → publie l'installeur + la Release.
Ensuite, chaque `npm run release` après un bump de version met à jour toutes les installations automatiquement.
