# Boucle #62 — Vague S.8 : trajet auto (adresse → heure de départ) · build 1.8.5

**Demande d'Adrien :** « Commence la Vague S.8 » → prolongement direct des détails d'agenda (#60). Choix explicite de l'approche : **OpenStreetMap sans clé**.

## Ce qui est livré

Tu renseignes une fois ton **point de départ** (adresse + mode : voiture / vélo / à pied). Ensuite, dans le formulaire d'ajout d'un événement, le bouton **🧭 Estimer** :
1. géocode ton départ **et** le 📍 lieu de l'événement (Nominatim / OpenStreetMap),
2. calcule l'itinéraire routier (OSRM),
3. remplit automatiquement le champ **🚗 trajet (min)** selon ton mode,
4. affiche « ✓ 32 min (🚗 voiture) · 12 km → destination ».

Ce trajet nourrit ensuite `departureInfo()` (#60) → **heure de départ conseillée** + « pars dans X min ».

## Modèle de sécurité (Vague S.8, non négociable)

- **Réseau uniquement dans le main** (`httpsGetJson`), renderer verrouillé inchangé.
- **Allowlist stricte par hôte** : `isAllowedTravelUrl` (pur + testé) n'autorise que `nominatim.openstreetmap.org` et `router.project-osrm.org`, en HTTPS, hôtes publics (réutilise `isPrivateHost`). Suffixe piégé `…openstreetmap.org.evil.com` **refusé**.
- Requêtes bornées (timeout 10 s, ≤ 2 Mo, ≤ 3 redirections https), **JSON parsé jamais exécuté**.
- **Opt-in strict** : rien ne sort tant qu'Adrien ne clique pas « Estimer ».
- **Point de départ chiffré** via `safeStorage` (`userData/travel.dat`), jamais en clair.
- **Sans clé / sans compte.**

## Détail technique

- `lib/logic.js` (purs + testés) : `TRAVEL_HOSTS`, `isAllowedTravelUrl`, `buildGeocodeUrl`, `buildRouteUrl` (ordre lon,lat), `haversineKm` (repli), `travelModes` (voiture = durée OSRM ; vélo 15 km/h, marche 5 km/h sur la distance routière).
- `electron-main.cjs` : `httpsGetJson` (même moule que `fetchIcs`), `geocodeAddress`, IPC `travel:estimate`, `travel:config:get/save` (safeStorage).
- `preload.cjs` : `estimateTravel` / `getTravelConfig` / `saveTravelConfig`.
- `index.html` : bouton `#calendarAgendaEstimate` + panneau repliable « 🧭 Point de départ » (`#travelStartForm`, `#travelHome`, `#travelMode`).
- `app.js` : `setupTravelStart()` (charge/enregistre la config), handler du bouton (statut via `textContent`, dégradé propre hors app installée).

## Vérifs

- **Flux réseau réel** vérifié (script hors Electron, mêmes builders) : Gare de Lorient → Gare de Rennes = **156,4 km**, voiture **123 min**, vélo 626 min. Géocodage + itinéraire + parsing OK.

## Limites / à venir

- Détection **GPS live** de « où je suis » : nécessiterait une clé de géolocalisation Chromium (désactivée par défaut dans Electron) → le point de départ manuel couvre déjà le besoin. Optionnel plus tard.
- Le géocodage en texte libre prend le meilleur résultat Nominatim : une adresse précise donne un meilleur match (le libellé affiché permet de vérifier).
- Reste en Vague S.8 (action d'Adrien) : **scan du frigo par photo** (clé d'API de vision), **sync agenda OAuth** (projet Google Cloud).
