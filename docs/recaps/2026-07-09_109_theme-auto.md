# Boucle #109 (autonome) — Thème auto (système) · build 1.9.43

**Contexte :** 34ᵉ itération de la boucle autonome. Aire : UX / apparence.

## Livré

Le bouton de thème ne fait plus un simple bascule clair/sombre : il **cycle sur 3 modes** — **auto (🌗) → clair (☀️) → sombre (🌙)**.

- En mode **auto**, l'app suit la préférence du système (`prefers-color-scheme`) et **bascule en direct** quand l'OS passe de clair à sombre (ou l'inverse).
- Le mode choisi est persistant ; l'infobulle du bouton indique le mode courant.
- Compatibilité ascendante : les valeurs `light`/`dark` déjà stockées restent respectées ; démarrage sans réglage = sombre (inchangé).

## Détail technique

- `lib/logic.js` : `nextThemeMode(current)` (cycle) et `resolveTheme(mode, systemDark)` — purs + testés.
- `app.js` :
  - IIFE d'amorçage réécrite pour résoudre le mode (dont auto) dès le chargement, sans flash.
  - `currentThemeMode` / `systemPrefersDark` / `applyTheme` + `applyThemeButton` (icône + titre selon le mode).
  - clic = `nextThemeMode`, et écoute `matchMedia('change')` pour rafraîchir en mode auto.
