# Boucle #236 (autonome) — 7ᵉ rotation #1 : badge d'app PWA · build 1.9.170

**7ᵉ rotation, #1 (mobile/PWA).** Rien ne ramenait l'utilisateur dans l'app une fois installée. Ajout d'un **badge sur l'icône PWA** (`navigator.setAppBadge`) avec le nombre d'actions en attente du jour — re-engagement discret.

## Livré

- **Pastille numérique sur l'icône installée** (écran d'accueil / barre des tâches) = **quêtes non faites + séances de sport planifiées aujourd'hui**.
- Mise à jour à chaque render ; **effacée automatiquement** quand tout est fait (0 → `clearAppBadge`).
- No-op silencieux là où l'API n'existe pas (navigateur non installé / desktop non compatible).

## Détail technique

- **`lib/logic.js`** : `pendingBadgeCount(state, todayKey)` — quêtes non cochées + agenda sport du jour, borné à 99. Pur + testé.
- **`app.js`** : `updateAppBadge()` (appelé dans `render()`) → `setAppBadge(n)` si n>0, sinon `clearAppBadge()`.

## Vérifs

- `npm run verify` → **268 tests / 268 pass** (+1 `pendingBadgeCount`), garde-fou CSS vert, **SMOKE OK** (`appBadge:true`).
- **Navigateur** (2 quêtes + 1 séance du jour) : `setAppBadge(3)` appelé ; après avoir tout coché → `clearAppBadge()`. ✓
- `npm run dist` → **Setup 1.9.170.exe** (app d'Adrien jamais fermée).
